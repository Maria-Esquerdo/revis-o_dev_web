Função e comportamento dos principais elementos semânticos HTML:

Elementos semânticos são tags específicas do HTML que têm função definida e propósito tanto para o navegador quanto para o programador.

Todos esses a seguir são elementos em bloco (block-level elements) por padrão. Isso significa que eles automaticamente começam em uma nova linha e ocupam 100% da largura disponível na tela, empilhando-se um em cima do outro.

Elementos em bloco podem ter elementos em bloco ou em linha como filhos.

Header:
Função: Atua como a introdução de uma página ou de uma seção específica. Ele reúne os elementos de identidade e caminhos globais do site.
Como se comporta: Fica posicionado no topo do contêiner onde foi inserido. Ele agrupa elementos visualmente destacados e geralmente estende-se por toda a largura horizontal para criar a "barra de topo" ou o cabeçalho de um artigo.
Nav:
Função: Centralizar e delimitar os blocos de links de navegação do site (menus). Os leitores de tela usam essa tag para permitir que usuários com deficiência visual pulem o menu e vão direto ao texto, ou acessem os links rapidamente.
Como se comporta: Geralmente fica dentro do header principal, mas também pode aparecer no
ou em barras laterais. Internamente, costuma abrigar uma lista ordenada ou não (
    ou
      ) com os links horizontais ou verticais.
Main:
Função: Representar o conteúdo exclusivo e central daquela página. É o indicador definitivo para o Google e navegadores sobre o que realmente importa naquele endereço.
Como se comporta: Funciona como o grande contêiner central da página. Por ser o "corpo de conteúdo", tudo o que muda de uma página para outra do seu site fica guardado dentro dele. Recomenda-se usar apenas um por documento.
Section
Função: Dividir o conteúdo em blocos temáticos ou capítulos. Use uma section quando os elementos internos estiverem muito relacionados e puderem ser agrupados sob um mesmo título de assunto.
Como se comporta: Ele empilha-se verticalmente dentro do
. É uma convenção de boas práticas que cada section comece com uma tag de título (h2 a h6) para dar um nome àquela parte do documento.
Article:
Função: Envolver um conteúdo que é totalmente autossuficiente e independente. Se você recortar um article do seu site e colá-lo em outro lugar, ele ainda fará sentido sozinho.
Como se comporta: Assim como as seções, ele cria um bloco vertical. É muito comum ver múltiplos article enfileirados ou organizados em grade (grid) dentro de uma section
Aside:
Função: Conter informações periféricas ou secundárias que complementam o assunto principal, mas não são obrigatórias para o entendimento (como termos relacionados, anúncios, biografias curtas ou widgets).
Como se comporta: Embora seja um elemento em bloco (que se empilha por padrão), no design visual e no CSS ele quase sempre é estilizado para ser posicionado como uma barra lateral (esquerda ou direita) ao lado do conteúdo do main ou do article.
Footer:
Função: Encerrar a página ou seção, trazendo dados institucionais, legais ou de fechamento.
Como se comporta: Fica rigidamente posicionado na base inferior da página (ou do contêiner onde foi inserido), estendendo-se horizontalmente para criar a faixa de rodapé do site.

No HTML temos um elemento, a div, que não comunica nada ao navegador, serve apenas como um container qualquer.

Div:
Serve para estruturar o layout, agrupar grandes pedaços de código ou criar "caixas" para organizar elementos dentro da página.
Ela sempre começa numa nova linha e ocupa 100% da largura disponível da página, empurrando qualquer coisa que venha depois para a linha de baixo.

Envolvem palavras/frases e ocupam apenas o espaço necessário para envolver os elementos definidos.

Permitem apenas elementos em linha como filhos.

Span:
Cria uma "seleção" invisível que envolve apenas o texto selecionado (horizontal/linha).
<span></span>
Tag de Link (link):
Localiza-se no head apenas, linka a página atual com outras páginas, para facilitar a navegação, guarda os links dos documentos das folhas de estilo da pagina ou javascript, por exemplo.
Possui os atributos href, que indica a URI do arquivo e o atributo rel, que define a relação do recurso adicionado com o documento atual.
Tag de Link (a):
Transforma um pedaço de texto em um link clicável sem quebrar o parágrafo, no body. Esse elemento se comporta como inline, mas pode marcar um bloco de texto também.
Exemplo: <p>Clique <a href="#">aqui</a> para baixar o arquivo.</p>
O atributo href recebe o URI de alguma página na internet. Podemos relacionar, também, com telefone e email.
Tags de Ênfase:
  • <strong></strong> -> Negrito
  • <i></i> -> Itálico
  • <mark></mark> -> Grifado
Tag de imagem (img):
Embora carregue um arquivo visual pesado, a tag <img> é, por padrão, um elemento inline! Se você colocar uma imagem pequena no meio de uma frase, ela vai se comportar como se fosse uma letra grande.
Exemplo: <p>Siga nosso perfil <img src="icone.png" alt="Insta"> no Instagram.</p>
o atributo src é o arquivo da imagem e o alt é o nome dela.
Tag de código(code):
<code> Código... </code>

Listas e Tabelas:

Listas não ordenadas
Listas indicadas por símbolos ondem a ordem dos elementos não é relevante.
Exemplo: 
                            
                                <ul>
                                    <li>Item 1</li>
                                    <li>Item 2</li>
                                    <li>Item 3</li>
                                </ul>
Listas ordenadas:
Listas numeradas, onde a ordem dos itens importa.
Exemplo: 
                            
                                <ol>
                                    <li>Item 1</li>
                                    <li>Item 2</li>
                                    <li>Item 3</li>
                                </ol>
Listas de Definição ou Descrição:
lista especializada usada para associar termos aos seus respectivos significados, descrições ou valores. Diferente das anteriores, ela não utiliza marcadores visuais comuns ou números, mas sim um recuo de margem (parágrafo avançado) nas descrições.
Exemplo: 
                            
                                <dl>
                                    <dt>Termo 1</dt>
                                        <dd>Definição.</dd>
                            
                                    <dt>Termo 2</dt>
                                        <dd>Definição.</dd>
Listas Aninhadas:
São uma combinação dessas listas. Uma lista dentro da outra.
Regra de sintaxe: A nova lista interna deve ser aberta obrigatoriamente dentro de uma tag <li>, e nunca diretamente dentro de um <ul> ou <ol>.
Exemplo: 
                            
                                <ul>
                                    <li>Tipo 1:
                                        <ul>
                                            <li> Item 1</li>
                                            <li>Item 2</li>
                                        </ul>
                                    </li>

                                    <li>Tipo 2: 
                                        <ul>
                                            <li> Item 1</li>
                                            <li>Item 2</li>
                                        </ul>
                                    </li>
                                </ul>
Tabelas:
Para dados que possuem relação bidimensional (linhas e colunas).
As tabelas HTML puras não vêm acompanhadas daquelas linhas de grade escuras ou bordas bonitas por padrão. Elas apenas alinham os textos perfeitamente. Para adicionar as linhas pretas divisórias tradicionais de uma tabela, você precisará aplicar a propriedade border ou estilos em CSS
Para montar uma tabela, utilizamos um conjunto de tags essenciais:
  • <table>: Define o início e o fim da tabela completa.
  • <thead>: Agrupa o conteúdo do cabeçalho da tabela (os títulos das colunas).
  • <tbody>: Agrupa o conteúdo principal ou o "corpo" com os dados da tabela.
  • <tfoot>: Agrupa o rodapé da tabela (usado geralmente para totais, resumos ou notas).
  • <tr> (Table Row): Cria uma linha na tabela. Tudo dentro de uma tabela precisa estar dentro de uma linha.
  • <th> (Table Header): Cria uma célula de cabeçalho. O texto fica automaticamente em negrito e centralizado pelo navegador.
  • <td> (Table Data): Cria uma célula comum de dados/conteúdo.
Atributos especiais para essas tags:
  • Colspan (Mesclar Colunas): Faz uma célula se esticar horizontalmente por mais de uma coluna.
  • Rowspan (Mesclar Linhas): Faz uma célula se esticar verticalmente por mais de uma linha.
Exemplo: 

                            <table>
                                <thead>
                                    <tr>
                                        <th scope="col1">Tipo 1</th>
                                        <th scope="col2">Tipo 2</th>  -> O atributo scope especifica o grupo de dados
                                        <th scope="col3">Tipo 3</th>
                                    </tr>
                                </thead>

                                <tbody>
                                    <tr>
                                        <td>Info 1.1</td>
                                        <td>Info 1.2</td>
                                        <td>Info 1.3</td>
                                    </tr>

                                    <tr>
                                        <td>Info 2.1</td>
                                        <td>Info 2.2</td>
                                        <td>Info 2.3</td>
                                    </tr>
                                </tbody>

                                <tfoot>
                                    <tr>
                                        <td>Final 1.1</td>
                                        <td>Final 1.2</td>
                                        <td>Final 1.3</td>
                                    </tr>
                                </tfoot>
                            </table>
